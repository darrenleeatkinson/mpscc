Viewport streaming — the scalability lever (§5.5)

The platform never streams 3k/sec to a browser. Instead:

  1. Map publishes bounds + zoom to locations.viewport-req (debounced ~250 ms).
  2. geo-query runs Redis GEOSEARCH BYBOX → snapshot (or clusters below zoom Z).
  3. Gateway forwards only locations.deltas whose geohash tile intersects the client's viewport — tens–hundreds/s, not thousands.
  4. Zoom-dependent detail: cluster bubbles below Z, typed icons with heading above Z.

Client load is bounded to what is visible, independent of the 40k fleet — the single most important scalability decision.
